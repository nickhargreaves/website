---
name: Git workflow preference
description: Nick prefers main-branch only workflow for simple/personal projects — no feature branches, no PRs
type: feedback
---

Work directly on the main branch for simple sites and personal projects. Do not create feature branches or PRs.

**Why:** Unnecessary overhead for a simple static website.
**How to apply:** Commit directly to main. Only suggest branches/PRs for larger, multi-contributor, or production-sensitive projects.
